State Machine

Invitation Workflow States

Multi-step item tracking pattern

sentItems
Athlete sent invitation — waiting for parent to accept. Shown in Settings → Sent Invitations (gray)
receivedItems
Parent received invite — sees call to action on join page. Shown as Received Invitations (blue)
pendingItems
Token validated, account creation in progress. Awaiting final confirmation step (amber)
completedItems
Parent joined family unit successfully. Both can now view and collaborate on recruiting data (green)

Security

Auth & Request Pipeline

Every API request goes through this chain

1
useAuthFetch (client)
Auto-injects session JWT and X-Request-ID correlation header into every $fetch call
composables/useAuthFetch.ts
2
Correlation Middleware
Reads X-Request-ID, attaches to all log entries for end-to-end request tracing
server/middleware/
3
requireAuth() + CSRF
Validates Supabase JWT. Mutation endpoints require CSRF token (double-submit cookie pattern)
server/utils/auth.tsuseCsrf.ts
4
useLogger(event, context)
Structured logging with correlation ID. Levels: error, warn, info, debug. Never exposes raw error messages.
server/utils/logger.ts
