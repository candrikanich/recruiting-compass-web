Auth Flow & Logging Strategy
Authentication pipeline and structured logging across all layers
🔐 Authentication Flow
1
Supabase Auth — user signs in via magic link or OAuth. Session JWT stored in browser.
2
useAuthFetch composable — wraps $fetch, auto-injects Authorization + X-Request-ID headers on every API call.
3
requireAuth() server util — validates JWT, returns { id, email }. Throws 401 if missing or expired.
4
family_unit_id resolution — server queries family_unit_members to derive the family context for RLS and access control.
5
Role-based routing — Nuxt middleware checks user.role (athlete / parent / coach) and redirects to appropriate onboarding or dashboard.
📊 Structured Logging
S
Server routes: useLogger(event, "context") — auto-captures correlation ID from X-Request-ID header.
C
Client utils: createClientLogger("context") — consistent prefix format for browser DevTools.
B
Background jobs: createLogger("cron/job-name") — no H3Event available; module-level instantiation.
📟 Server Log Levels
error — DB failures, unhandled exceptions
warn — data not found, business rule blocks
info — request processed, action completed
debug — verbose, dev-only context
🔗 Correlation Flow
client session → X-Request-ID
useAuthFetch → header injection
correlation mw → reads + attaches
useLogger(evt) → every entry tagged
